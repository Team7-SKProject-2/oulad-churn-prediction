# 프로젝트 요구사항

## 사용자

- 교육 서비스 운영 담당자
- 학습 상담 담당자
- 프로젝트 분석·모델링 담당자

## 문제 정의

OULAD 학습자의 초기 행동을 이용해 최종 중도이탈 여부를 예측하고, 위험 학생에게 적용할 유지 활동을 제안한다.

## 분석 단위와 Target

- 분석 단위: 학생 × 과목 × 강좌 개설 회차
- 이탈: `final_result == "Withdrawn"`
- 비이탈: `Pass`, `Distinction`, `Fail`
- 출력: 이탈 확률, 위험 등급, 주요 위험 신호, 추천 유지 활동

## 골든타임

1·2·4주차 누적 데이터의 Validation 성능을 비교한다. 실제 이탈자를 충분히 찾아내는 시점 중 가장 빠른 주차를 골든타임으로 선정한다.

## 평가 지표

- 주지표: Recall
- 보조지표: F1, Precision, PR-AUC, ROC-AUC
- Accuracy만으로 모델을 선택하지 않는다.

## 완료 조건

- 원본 데이터 수정 없이 재현 가능한 전처리 코드가 있다.
- 동일 학생이 데이터 분할 사이에 중복되지 않는다.
- 1·2·4주차 각 시점 이후 정보가 해당 스냅샷에 포함되지 않는다.
- 과목×주차 이탈률과 과목별 행동 차이를 확인할 수 있다.
- 위험 신호와 추천 유지 활동이 함께 출력된다.
- 최종 모델과 전처리 파이프라인을 하나로 저장한다.
- Streamlit이 저장된 모델을 실제로 불러와 추론한다.
- README, 전처리 보고서, 학습 보고서, 발표 자료를 완성한다.
